# Our proposed method:
This implementation performs an iterative batch-correction and clustering procedure for multivariate feature-by-sample data by alternating soft clustering updates with a mixture-of-experts ridge correction to remove batch-associated signals. On input it accepts a feature-by-sample data matrix, a per-sample categorical batch design encoded as indicator matrices and priors, per-cluster scaling parameters and hyperparameters (regularization, multiplicative damping exponents, cluster count, block update size), iteration limits, convergence tolerances, verbosity and an optional random seed or clustering initializer; it outputs a corrected data matrix of the same shape. The routine first standardizes features by z-score and normalizes sample vectors to unit length, then initializes cluster centroids via a k-means procedure and computes soft cluster responsibilities by converting cosine distances into a stabilized softmax; batch- and cluster-level sufficient statistics are accumulated and an objective decomposed into k-means error, an entropy regularizer, and a cross-entropy batch-enrichment term. Iteration alternates between (1) refining cluster centroids and responsibilities via repeated centroid updates and block-wise multiplicative responsibility updates that incorporate batch prior ratios and exponentiated log-enrichment weights while shuffling update order for stochasticity and maintaining L1 normalization, and (2) applying a per-cluster ridge regression correction on the original standardized data to estimate and subtract batch-associated contributions (with explicit exclusion of the intercept from removal), followed by renormalization of corrected vectors; numerical safeguards include clamping tiny variances and norms to avoid division by zero, stabilizing exponentiations and logarithms by thresholding and zeroing non-finite terms, and using ridge regularization to ensure invertibility. Convergence is detected using relative changes of the tracked objective over a sliding window for the clustering inner loop and a separate relative-change criterion for the outer harmony loop; diagnostic quantities (full objective and its components, number of k-means rounds) are recorded for monitoring, and the corrected standardized data matrix is returned.
# Baseline method:

- harmony method
```
This implementation performs iterative harmonization of high-dimensional sample measurements by alternating soft clustering and cluster-wise regression-based covariate removal. Inputs consist of a feature-by-sample data matrix, a categorical batch indicator matrix, a covariate design matrix for mixture-of-experts correction, prior batch frequencies, a per-cluster scale vector and batch-weight vector, regularization and algorithmic hyperparameters (number of clusters, block size, convergence tolerances, maximum iterations), and an optional random seed for initialization. Processing begins with per-sample cosine normalization and k-means initialization of cluster centroids on the normalized data. Soft cluster responsibilities are computed from cosine-based pairwise distances scaled by per-cluster scales and converted to normalized probabilities via a numerically stabilized exponential transform; responsibilities are further modulated multiplicatively by batch-diversity factors derived from log-ratios of observed versus expected batch counts weighted by the batch-weight vector. The inner loop alternates centroid updates (as projections of normalized samples by responsibilities) with responsibility updates; responsibility updates are performed in randomized blocks to incrementally remove and reinsert subsets of samples while maintaining running expected- and observed-batch statistics. After each clustering phase, an outer step performs cluster-wise ridge regression of the original data against the covariate design using responsibilities as weights, subtracts the fitted covariate contributions (preserving the intercept) to produce a corrected data matrix, and renormalizes the corrected data for the next iteration. The implementation computes and records a composite objective composed of weighted clustering distance, an entropy penalty (computed with a safe-log mechanism to handle zeros and non-finite values), and a batch cross-entropy term; these diagnostics drive windowed relative-change convergence tests for both the inner clustering loop and the outer harmonization loop. Special numerical safeguards include log-sum-exp style subtraction prior to exponentiation, explicit finite-value handling in entropy computation, additive smoothing in batch log-ratios, L2 regularization in ridge solves, block-wise stochastic updates for scalability, and preservation of the regression intercept; the routine returns the covariate-corrected data matrix and retains diagnostic histories for inspection.
```
